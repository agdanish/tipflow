# Hackathon Mechanics

## Key Facts

- **Name:** Tether Hackathon Galactica: WDK Edition 1
- **Platform:** DoraHacks
- **Format:** Virtual, online
- **Period:** Feb 25 – Mar 22, 2026
- **Submission deadline:** March 22, 2026 23:59 UTC (~6 days from now)
- **Registrants:** 324 | **Submissions:** 8 (as of Mar 16)

## Prize Distribution (30,000 USDT total)

**Best Overall:**
- 1st: 6,000 | 2nd: 3,000 | 3rd: 1,000

**Track Prizes (4 tracks, each 5,000 USDT):**

| Track | 1st | 2nd | Current Submissions |
|-------|-----|-----|---------------------|
| Agent Wallets (WDK/Openclaw) | 3,000 | 2,000 | ~4 (AMP, Axiom, Tsentry, peaq, Ajo-Agent) |
| Lending Bot | 3,000 | 2,000 | ~1 (LendGuard) |
| Autonomous DeFi Agent | 3,000 | 2,000 | ~2 (PayMind AI, SafeAgent) |
| Tipping Bot | 3,000 | 2,000 | **0** |

## Submission Requirements

1. Product name + brief description
2. Track selection
3. Team members with background
4. Team location
5. **Public GitHub repo** (Apache 2.0 license)
6. **Technical overview video** (YouTube unlisted, max 5 min)
   - Must include project overview + product demo
   - Clear, concise, high quality

## Technical Requirements

- Must integrate WDK by Tether (JS/TS) meaningfully
- Proper agent architecture + WDK wallet integration mandatory
- Clear setup/run instructions
- Disclose all third-party services/APIs
- Easy for judges to evaluate (web browser accessible or runnable out-of-box)

## Rules

- Single entry only
- No dual participation (solo + team)
- Pre-existing code allowed but must be disclosed
- Apache 2.0 license required
- Work during hackathon period is what gets judged

## Judging Criteria (7 categories from Rules page)

1. **Agent Intelligence** — LLM use, autonomous decision-making, real actions
2. **WDK Wallet Integration** — Secure, correct, non-custodial, robust transactions
3. **Technical Execution** — Architecture, code quality, reliability
4. **Agentic Payment Design** — Realistic programmable payment flows
5. **Originality** — Innovative use case, creative agent-wallet interaction
6. **Polish & Ship-ability** — Completeness, UX clarity, deployment readiness
7. **Presentation & Demo** — Clear explanation + strong live demo

**Key quote from judges:** "Strong architecture, intelligent agent behavior, and sound wallet implementation matter more than surface-level UI."
